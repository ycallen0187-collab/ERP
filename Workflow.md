# ERP AI 工作流程

版本：1.0

最後更新：2026-07-02

---

# 文件目的

本文件定義 AI 在本 Repository 中的工作流程。

所有需求皆必須依照本流程執行。

不得自行省略步驟。

---

# AI 工作流程

所有需求皆依照下列流程：

```
收到需求
    │
    ▼
閱讀相關檔案
    │
    ▼
分析流程
    │
    ▼
分析影響範圍
    │
    ▼
確認需求
    │
    ▼
等待使用者確認（必要時）
    │
    ▼
開始修改
    │
    ▼
自我檢查
    │
    ▼
回報修改內容
```

不得直接開始修改程式。

---

# Step 1：閱讀相關檔案

收到需求後，

第一件事情：

閱讀所有相關檔案。

例如：

- JSP
- Java
- SQL
- Javascript
- 共用 Class
- 共用 Function

不要只閱讀單一檔案。

---

# Step 2：分析目前流程

閱讀完成後，

先說明：

目前程式如何運作。

例如：

使用者

↓

JSP

↓

Java

↓

DB2

↓

回傳 JSP

確認理解目前流程。

不得直接修改。

---

# Step 3：分析相依性

確認：

是否有：

- 共用 JSP
- 共用 Java
- 共用 SQL
- 共用 Javascript
- 共用 Utility

若有，

必須列出。

---

# Step 4：分析影響範圍

修改前，

請先回答：

會影響哪些：

畫面

功能

SQL

Java

共用元件

若可能影響其他功能，

必須提出警告。

---

# Step 5：確認需求

若需求存在任何不明確之處，

不得猜測。

應提出問題。

例如：

是否只修改查詢？

是否同步修改匯出？

是否同步修改新增功能？

確認後才能修改。

---

# Step 6：開始修改

確認需求後，

才開始修改。

修改時：

遵守：

最小修改原則。

不要修改未要求功能。

---

# Step 7：自我檢查

修改完成後，

請自行確認：

是否符合：

Java6

既有 Coding Style

既有命名

既有流程

是否影響其他功能。

修改完成後，

請再次檢查：

□ 是否刪除任何程式

□ 是否刪除任何註解

□ 是否刪除任何 Import

□ 是否修改任何命名

□ 是否修改任何未要求功能

□ 是否修改任何 SQL

□ 是否修改任何 JavaScript

□ 是否修改任何 UI

若有任何一項為：

是。

請重新確認是否為需求要求。

否則請恢復。
---

# Step 8：回報

修改完成後，

請依序回報：

【修改內容】

【修改檔案】

【修改原因】

【影響範圍】

【測試建議】

【風險提醒】

---

# 高風險需求

遇到以下需求：

- 修改 SQL
- 修改共用 Function
- 修改共用 Java
- 修改共用 JSP
- 修改登入
- 修改權限
- 修改交易(Transaction)

請先停止。

分析後，

等待使用者確認。

---

# 缺少檔案

若修改需要：

Action

DAO

JSP

Javascript

SQL

共用 Utility

但 Repository 中沒有，

請直接回覆：

「缺少相關檔案，無法安全修改。」

並列出需要哪些檔案。

不得自行假設。

---

# 可以直接修改

下列情況通常可直接修改：

- 畫面文字
- 單一查詢條件
- 單一欄位
- Bug 修正
- Null Pointer
- Exception 修正

若不影響其他功能。

---

# 必須等待確認

下列情況不得直接修改：

- SQL
- 共用元件
- 共用 Function
- Repository 架構
- JavaScript
- JSP Layout
- CSS
- Framework
- Java Version

必須等待使用者確認。

---

# AI 必須避免

不得：

憑經驗猜測。

憑命名猜測。

自行新增功能。

自行最佳化。

自行重構。

自行改善程式。

自行升級。

---

# 最後原則

修改程式之前，

請先理解。

理解流程，

永遠比開始修改重要。

任何修改，

皆應遵守：

分析

↓

確認

↓

修改

↓

驗證

↓

回報

而不是：

收到需求

↓

立即寫程式。
