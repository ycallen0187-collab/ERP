# OutputFormat.md

# AI 回覆格式規範

版本：1.0

最後更新：2026-07-02

---

# 文件目的

本文件定義 AI 修改程式時的回覆格式。

所有 AI 回覆皆應遵守。

---

# 第一原則

不要長篇理論。

不要介紹技術。

不要解釋教科書內容。

請直接回答重點。

---

# 修改前

修改任何程式前，

請依照下列格式：

--------------------------------

【需求理解】

(一句話說明理解內容)

【修改檔案】

- xxx.jsp
- xxx.java

【影響範圍】

- 查詢功能
- 匯出功能

【風險】

- 無

【需要確認】

- 無

--------------------------------

---

# 修改後

修改完成後，

請依照下列格式：

--------------------------------

【修改完成】

【修改檔案】

- xxx.jsp
- xxx.java

【修改內容】

1.

2.

3.

【修改原因】

(一句話即可)

【測試建議】

1.

2.

【風險提醒】

無

【AI 自我檢查】

請確認：

□ 未刪除任何既有程式

□ 未刪除任何既有註解

□ 未刪除任何 Import

□ 未修改任何命名

□ 未修改任何 UI

□ 未修改任何 JavaScript

□ 未修改任何 SQL

□ 僅修改需求指定範圍

--------------------------------

---

# Java

若修改 Java：

請提供：

完整 Java 檔案。

不要只提供修改片段。

---

# JSP

若修改 JSP：

請提供：

完整 JSP。

不要只提供修改區塊。

---

# SQL

若修改 SQL：

請提供：

完整 SQL。

並說明：

是否需要：

- RUNSTATS
- REORG

---

# 多個檔案

若修改多個檔案，

請全部提供。

不要只提供其中一個。

---

# 缺少檔案

若缺少：

Java

JSP

SQL

Javascript

共用 Class

請直接回答：

--------------------------------

缺少以下檔案：

1.

2.

3.

目前無法安全修改。

--------------------------------

不要自行假設。

---

# 發現風險

若發現：

共用程式。

共用 SQL。

共用 JSP。

請直接回答：

--------------------------------

【風險提醒】

此次修改可能影響：

1.

2.

3.

建議先確認。

--------------------------------

---

# 發現更好的寫法

若發現：

SQL

Java

架構

可改善。

請不要直接修改。

請最後增加：

--------------------------------

【改善建議】

(若需要，我可以另外提供改善版本。)

--------------------------------

不要直接套用。

---

# 回覆原則

請保持：

簡潔。

直接。

有重點。

不要長篇說明。

不要重複描述需求。

不要介紹理論。

若能直接完成，

就直接完成。

---

# 最後原則

回答以：

完成需求

為第一優先。

分析只是輔助。

不要讓使用者一直等待。
